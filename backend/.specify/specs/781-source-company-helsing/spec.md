## 1. Problem Statement

Helsing is a European defense technology company that develops
artificial intelligence software to process sensor and weapon-system
data for military and national-security use. Its products span AI
for intelligence, surveillance, and reconnaissance, electronic
warfare, and autonomous systems including strike drones and
aircraft. Founded in 2021, the company operates across Germany, the
UK, and France, serving European defense ministries and armed
forces. It is among Europe's most highly valued defense-tech
startups.

Helsing publishes its careers board through Greenhouse at
the bare slug `helsing`. The run-398 batch sweep confirmed
134 live role(s) via a direct probe of
`https://api.greenhouse.io/v1/boards/helsing/jobs?content=true`.
Sector: Defense Tech / AI. HQ: Munich, Bavaria, Germany.

## 2. Goals

- Ship a `source-company-helsing` plugin returning live
  `JobPostDto` rows from the Greenhouse board.
- Mirror the canonical variant-2 + D-08 company-direct template
  (wire `company_name` pass-through; defensive title/department
  trim; entity-decode-then-tag-strip descriptions).
- Bundle a unit-test suite (≥ 9 cases) with a mocked HTTP fixture.
- Publish the plugin's `Module` in `ALL_SOURCE_MODULES`.

## 3. Non-Goals

- Authenticated Greenhouse Harvest API support.
- Backfilling historical Helsing postings.
- Cross-board enrichment or salary inference.

## 4. User / Caller Stories

> As an **aggregator caller**, I want **`Site.HELSING`** in the
> source registry, so that **a single `siteType: [Site.HELSING]`
> request returns Helsing's open roles**.

## 5. Functional Requirements

| ID    | Requirement                                                                 | Priority |
| ----- | --------------------------------------------------------------------------- | -------- |
| FR-1  | Add `Site.HELSING = 'helsing'` to the `Site` enum.               | must     |
| FR-2  | New plugin package `@ever-jobs/source-company-helsing`.                  | must     |
| FR-3  | `HelsingService.scrape(input)` returns a `JobResponseDto`; never throws. | must   |
| FR-4  | The plugin is registered in `ALL_SOURCE_MODULES`.                          | must     |
| FR-5  | `tsconfig.base.json` path-alias + matching `jest.config.js` mapper.      | must     |
| FR-6  | Each `JobPostDto` `id` prefixed `helsing-`, `site === Site.HELSING`. | must |
| FR-7  | `input.resultsWanted` honoured.                                           | must     |
| FR-8  | `input.searchTerm` honoured (title + department substring).               | should   |
| FR-9  | Network errors caught — returns `{ jobs: [] }`.                            | must     |
| FR-10 | ≥ 9 unit tests with mocked HTTP.                                            | must     |
| FR-11 | D-08 decode-then-strip pipeline.                                            | must     |
| FR-12 | Wire `absolute_url` flows through to `jobUrl` (variant 2).               | must     |
| FR-13 | D-10 defensive `.trim()` on wire titles.                                   | must     |
| FR-14 | D-11 defensive `.trim()` on wire department names.                         | must     |

## 6. Non-Functional Requirements

NFR-1..NFR-4 — same as cohort baseline (no new runtime deps;
`Promise`-based async; `Logger` not `console`; resilient to
malformed payloads).

## 7. Contracts

### 7.1 API / Interface

```ts
@SourcePlugin({ site: Site.HELSING, name: 'Helsing', category: 'company' })
@Injectable()
export class HelsingService implements IScraper {
  scrape(input: ScraperInputDto): Promise<JobResponseDto>;
}
```

### 7.2 Errors

All transport errors swallowed; caller sees `{ jobs: [] }`.

## 8. Test Plan

- ≥ 9 cases. Happy-path asserts variant-2 URL pass-through
  (`job-boards.greenhouse.io/helsing/jobs/<id>`), wire
  `company_name` pass-through (`'Helsing'`),
  D-10 title-trim lock, D-11 department-trim lock, and the
  D-08 decode-then-strip regression guard.
- Plus standard cohort cases (resultsWanted cap, searchTerm
  match + non-match, HTTP 500 error handling, empty payload).
