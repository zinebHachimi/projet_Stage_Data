## 1. Problem Statement

Jumio is an identity verification and compliance technology company
that provides AI-powered solutions for verifying users' identities,
assessing risk, and meeting KYC (Know Your Customer) and AML
(Anti-Money Laundering) regulatory requirements. Its platform
combines document verification, biometric facial recognition with
liveness detection, and machine learning to help businesses onboard
customers and prevent fraud. Jumio serves customers across financial
services, fintech, gaming, cryptocurrency, and other regulated
industries worldwide.

Jumio publishes its careers board through Greenhouse at
the bare slug `jumio`. The run-398 batch sweep confirmed
18 live role(s) via a direct probe of
`https://api.greenhouse.io/v1/boards/jumio/jobs?content=true`.
Sector: Identity & Fraud. HQ: Palo Alto, California, USA.

## 2. Goals

- Ship a `source-company-jumio` plugin returning live
  `JobPostDto` rows from the Greenhouse board.
- Mirror the canonical variant-2 + D-08 company-direct template
  (wire `company_name` pass-through; defensive title/department
  trim; entity-decode-then-tag-strip descriptions).
- Bundle a unit-test suite (≥ 9 cases) with a mocked HTTP fixture.
- Publish the plugin's `Module` in `ALL_SOURCE_MODULES`.

## 3. Non-Goals

- Authenticated Greenhouse Harvest API support.
- Backfilling historical Jumio postings.
- Cross-board enrichment or salary inference.

## 4. User / Caller Stories

> As an **aggregator caller**, I want **`Site.JUMIO`** in the
> source registry, so that **a single `siteType: [Site.JUMIO]`
> request returns Jumio's open roles**.

## 5. Functional Requirements

| ID    | Requirement                                                                 | Priority |
| ----- | --------------------------------------------------------------------------- | -------- |
| FR-1  | Add `Site.JUMIO = 'jumio'` to the `Site` enum.               | must     |
| FR-2  | New plugin package `@ever-jobs/source-company-jumio`.                  | must     |
| FR-3  | `JumioService.scrape(input)` returns a `JobResponseDto`; never throws. | must   |
| FR-4  | The plugin is registered in `ALL_SOURCE_MODULES`.                          | must     |
| FR-5  | `tsconfig.base.json` path-alias + matching `jest.config.js` mapper.      | must     |
| FR-6  | Each `JobPostDto` `id` prefixed `jumio-`, `site === Site.JUMIO`. | must |
| FR-7  | `input.resultsWanted` honoured.                                           | must     |
| FR-8  | `input.searchTerm` honoured (title + department substring).               | should   |
| FR-9  | Network errors caught — returns `{ jobs: [] }`.                            | must     |
| FR-10 | ≥ 9 unit tests with mocked HTTP.                                            | must     |
| FR-11 | D-08 decode-then-strip pipeline.                                            | must     |
| FR-12 | Wire `absolute_url` flows through to `jobUrl` (variant 2).               | must     |
| FR-13 | D-10 defensive `.trim()` on wire titles.                                   | must     |
| FR-14 | D-11 defensive `.trim()` on wire department names.                         | must     |

## 6. Non-Functional Requirements

NFR-1..NFR-4 — same as cohort baseline (no new runtime deps;
`Promise`-based async; `Logger` not `console`; resilient to
malformed payloads).

## 7. Contracts

### 7.1 API / Interface

```ts
@SourcePlugin({ site: Site.JUMIO, name: 'Jumio', category: 'company' })
@Injectable()
export class JumioService implements IScraper {
  scrape(input: ScraperInputDto): Promise<JobResponseDto>;
}
```

### 7.2 Errors

All transport errors swallowed; caller sees `{ jobs: [] }`.

## 8. Test Plan

- ≥ 9 cases. Happy-path asserts variant-2 URL pass-through
  (`job-boards.greenhouse.io/jumio/jobs/<id>`), wire
  `company_name` pass-through (`'Jumio'`),
  D-10 title-trim lock, D-11 department-trim lock, and the
  D-08 decode-then-strip regression guard.
- Plus standard cohort cases (resultsWanted cap, searchTerm
  match + non-match, HTTP 500 error handling, empty payload).
